  <Accordion title="What's the general syntax of Nx commands?">
  <img src="https://nx.dev/documentation/shared/images/run-target-syntax.svg" />

To execute tasks with Nx use the following syntax:

```sh title="Terminal"
pnpm nx run <project>:<target> <...options>
```

You can also run multiple targets:

```sh title="Terminal"
pnpm nx run-many -t <target1> <target2>
```

..or add `-p` to filter specific projects

```sh title="Terminal"
pnpm nx run-many -t <target1> <target2> -p <proj1> <proj2>
```

Targets can be defined in the `package.json` or `projects.json`. [Learn more](https://nx.dev/features/run-tasks).

Explore the project graph to see the projects in the monorepo and the tasks you can run with Nx.

```sh title="Terminal"
pnpm nx graph
```

[Learn more](https://nx.dev/core-features/explore-graph).

Use [Nx Console](https://nx.dev/nx-console) to integrate with editors.

<img src="https://nx.dev/documentation/shared/images/nx-console/nx-console-screenshot.webp" />

### Build for production

Run `pnpm nx run portal:build` to build the application. The build artifacts are stored in the output directory (e.g. `dist/` or `build/`), ready to be deployed.

### Set up CI

Nx comes with local caching already built-in (check your `nx.json`). On CI you might want to go a step further.

- [Set up remote caching](https://nx.dev/features/share-your-cache)
- [Set up task distribution across multiple machines](https://nx.dev/nx-cloud/features/distribute-task-execution)
- [Learn more how to setup CI](https://nx.dev/recipes/ci)
