`--verbose` provides detailed error logs.

Explore the project graph to see the projects in the monorepo and the tasks you can run with Nx.

```sh title="Terminal"
pnpm nx graph --verbose
```

<Accordions>
  <Accordion title="What's the general syntax of Nx commands?">
  <img src="https://nx.dev/documentation/shared/images/run-target-syntax.svg" />

You can also run multiple targets:

```sh title="Terminal"
pnpm nx run-many -t <target1> <target2>
```

..or add `-p` to filter specific projects

```sh title="Terminal"
pnpm nx run-many -t <target1> <target2> -p <proj1> <proj2>
```

Targets can be defined in the `package.json` or `projects.json`. [Learn more](https://nx.dev/features/run-tasks).

[Learn more](https://nx.dev/core-features/explore-graph).

Use [Nx Console](https://nx.dev/nx-console) to integrate with editors.

You'll see the url(s) in your terminal for the project(s) you're running, simply paste those in to your browser address bar.

    For example you may see:

```sh title="Terminal"
╰─ pnpm nx run docs:dev

> pnpm nx run docs:dev

> next dev

▲ Next.js 14.2.3

- Local: <http://localhost:3000>
✓ Starting...
automatically enabled Fast Refresh for 1 custom loader
```

As the project is in its early stages, these URLs may change. If you have multiple apps running or have another server using that network port, yours may be different from someone else's and that's okay.

Always look at your terminal for the correct URL.
